Platform Migration Plan — Q3 2026

Overview: Migrate core payment processing from legacy monolith (Java 8) to distributed microservices architecture (Go + gRPC). Target completion: September 30, 2026.

Phase 1 — API Gateway Cutover (July 1-15):
- Deploy Envoy sidecar proxies to production fleet
- Route 5% canary traffic through new gateway
- Fallback: instant revert via feature flag PF-2891
- Risk: 200ms latency increase during dual-write period

Phase 2 — Database Sharding (July 16 - August 15):
- Shard customer_transactions table by region (US-WEST, US-EAST, EU, APAC)
- Estimated downtime: 45 minutes during maintenance window
- Data migration tool: internal fork of gh-ost (modified for multi-region)
- Rollback: restore from hourly WAL snapshots

Phase 3 — Legacy Decommission (August 16 - September 30):
- Sunset Java monolith after 30-day parallel-run verification
- Archive source to internal GitLab cold storage
- Decommission 48 bare-metal servers in us-east-1 (annual savings: $840,000)

Security Considerations:
- All inter-service traffic encrypted via mTLS (cert rotation every 72 hours)
- New service mesh requires SOC 2 Type II re-audit (scheduled August 20)
- PCI DSS scope reduction expected post-migration (fewer systems handling card data)
